#前言
符号表是实现查找的数据结构  
符号表的目的是将一个键和一个值联系起来  
定义:是一种存储键值对的数据结构，支持两种操作:插入(put),即将一组新的键值对存入表中，查找(get)，即根据给定的键得到相应的表。  

特征:  
1. 每个只对应着一个值  
2. 当用例代码向表中存入键值对和表中已有的键冲突时，新的键会替代旧的值。  

符号表的核心是如何实现高效的查找和插入  

初级的符号表  
1. 无序链表的顺序查找  
2. 有序数组中的二分查找  

高效的符号表  
1. 二叉查找树  
2. 红黑树  
3. 散列表  

#顺序查找(基于无序链表)  
思路:get()的实现即为遍历链表，用equals()方法比较需要被查找的键和每个节点中的键，如果匹配成功我们就返回相应的值，否则我们返回null,pull()的实现也是遍历链表，用equals()方法比较需被查找的键和每个结点中的键。如果匹配成功我们就用第二个参数指定的值更新和该键想关联的值，否则创建一个新的结点并将其插入到链表的开头。
