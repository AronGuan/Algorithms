#基于线性探测法的散列表  
使用一对平行数组，一条保存键，一条保存值，使用散列函数产生访问数组所需的数组索引  
思路：用大小为M的数组保存N个键值对，其中M>N,需要依靠数组中的空位解决碰撞冲突，  
线性探测：当碰撞发生时(当一个键的散列表已经被另一个不同的键占用)，直接检查散列表中的下一个位置(将索引值加1)  

删除操作  
直接将改键所在的位置设为null是不行的。因此需要将簇中被删除键的右侧的所有键重新插入散列表。  

键簇：元素在插入数组后聚集成的一组连续的条目。  
带来的缺点，当散列表快满的时候查找所需的探测次数时巨大的,  所以要调整数组大小。  

散列表支持和数组大小无关的常数级别的查找和插入操作
