#基于三向单词查找树  
目的:避免R向单词查找树过度的空间消耗  
特点:每个结点都含有一个字符，三条链接和一个值，这三条链接分别对应着当前字母小于，等于和大于结点的所有键。  

查找和插入  
查找  
首先比较键的首字母和根结点的字母，如果键的首字母较小，就选择左链接，如果较大，就选择右链接，如果相等，则选择中链接。然后递归地使用相同的算法，如果遇到了一个空链接或者当键结束时结点的值为空，那么查找未命中，如果键结束时结点的值非空则查找命中。  
插入  
首先进行查找，在树中补全键某尾的所有结点。  

性质  
和其他所有二叉查找树一样，每个单词查找树结点的二叉查找树表示也取决于键的插入顺序。  
并且与R向单词查找树相比，字符是显示地保存在结点中的――――只有沿着中间链接前进时才会根据字符找到表中的键。
